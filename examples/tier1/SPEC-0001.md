# SPEC-0001: Welcome Email on Registration
intent_ref: INT-0001
status: active
domain: notifications
prior_art: null

## Behavior
- Registration handler calls `notifications/sender.ts#sendWelcome(userId)` after user record is committed
- `sendWelcome` is non-blocking (async, no await at call site)
- Email includes: user first name, confirmation link (`/verify?token={token}`)
- Confirmation token: 64-char hex, 24h TTL, stored in `db/tokens.ts`
- On send failure: log error, do not surface to user, do not retry synchronously

## Acceptance Criteria
- [ ] Welcome email dispatched within 60s of successful registration
- [ ] Registration endpoint returns 201 before email is sent (non-blocking verified)
- [ ] Email body contains user first name and a valid /verify link
- [ ] Failed send logs error with userId and does not throw to caller

## Impact Zones
- auth/registration.ts (MODIFIED · HIGH)
- notifications/sender.ts (MODIFIED · HIGH)
- db/tokens.ts (MODIFIED · MEDIUM)
- Upstream: user creation flow · Downstream: email delivery queue

## Out of Scope
- Email template changes · SMS/push variants · Unsubscribe flow
